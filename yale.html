Technical Projects

Fedora Commons Repository

  • Contributed to Fedora 4 features (such as policy driven storage), participated in Scrum sprints, and fixed bugs. An export of some JIRA tickets is available here, and so is an export of Pivotal Tracker tickets. More details can be found in Pivotal and JIRA.

  • Contributed to project documentation. Some of the pages I haved added or edited are listed here.

(Web) Services

  • Created several JAX-RS based services such as an archival content export service (for ArchivesSpace). The export service conducts change analysis to determine if the relevant metadata has changed since the last export. The service also applies several stylesheets to validate and generate MODS.
  • Created a SOAP-based persistent identifier management service that was used by many departments. The service allowed machine-to-machine bulk handle creation, and a JSP/Servlets-based GUI was also created to facilitate testing and spreadsheet export. This service was one of the very first projects launched by the Office of Digital Assets and Images, and it was critical for this project to succeed. Millions of handles were created by his service (e.g., by YUL). A a newer improved version of this service exists (created by aother team).
  • Created a central logging service to help build error visualizations (for apps like Tableau). After experimenting with ELK, I introduced Splunk as an alternative to this service.
  • Helped create an application for exporting the contents of a large storage repository. The service exported content using the BagIt format and relied on a MySql database to track progress. JAXB was used for marshalling/ unmarshalling XML/objects from metadata schemas such as MODS and CDWA Lite.

Image Collections

  • Created Palmquist (developed w/ Spring Boot/MVC/JPA), launched RAAI, and enabled audio streaming using QuickTime, RTSP and MacOS X Server for an international collection (now offline).
  • Contributed performance improvements to prominent Hydra-based collections.
  • Analyzed Solr configuration (schema, core setup) and customized it to improve search results for non-Latin scripts. (This project is in progress.)
  • Analyzed Solr configuration to identify potential performance bottlenecks. I found out that we could not balance our Solr ingest and query speeds well because of the particular gem versions/Solr we were using.

Enterprise Systems

  • Before using Canvas, Yale used to heavily rely on Sakai as the primary learning management system. I helped extend the Java-based Sign-up tool, a core Sakai tool that helps organize meetings. I also created services to export content and change forum permissions. Modifying these tools and services involved working with the Sakai API, JSF, JAX-WS, Spring and Hibernate.
  • Maintained SharePoint (used by staff for workflows and content management) intermittently. Successfully installed and migrated a complex enterprise instance from a failing server to a new server.

Staff Tools

  • Created a web application that helps staff manage inventory (students scan and report books that are shelved out of order, etc.). This application is used by many Yale libraries (and perhaps even at UConn).
